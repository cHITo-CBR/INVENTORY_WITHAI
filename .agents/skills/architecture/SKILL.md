# Proper Architecture Skill

This skill ensures that the codebase follows established software engineering patterns for long-term maintainability.

## Core Principles
1. **SOLID Principles**: Single Responsibility, Open/Closed, Liskov Substitution, Interface Segregation, Dependency Inversion.
2. **Clean Architecture**: Organize code into layers where dependencies point inward toward the business logic.
3. **DRY (Don't Repeat Yourself)**: Abstract common patterns while avoiding premature optimization.
4. **YAGNI (You Ain't Gonna Need It)**: Focus on current requirements while keeping the system extensible.

## Best Practices
- Use Dependency Injection where needed to improve testability.
- Keep components small and focused.
- Document architectural decisions (e.g., ADRs).

## Verification
- Code reviews should focus on architectural alignment.
- Use static analysis tools to enforce boundaries between layers.
