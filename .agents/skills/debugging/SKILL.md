# Debugging Proficiency Skill

This skill outlines the methodologies for identifying, isolating, and fixing software defects.

## Core Principles
1. **Root Cause Analysis (RCA)**: Don't just fix the symptom; find the underlying cause.
2. **Systematic Isolate**: Use binary search, logging, and simplified test cases to narrowing down the issue.
3. **Reproducibility**: Always create a minimal reproducible example (repro) before attempting a fix.
4. **Tool Mastery**: Proficient use of browser DevTools, IDE debuggers, and terminal logs.

## Best Practices
- Read error messages carefully—they often tell you exactly what's wrong.
- Verify assumptions at every step.
- Use unit tests to prevent regressions after a fix.

## Methodology
- Observe: Identify the unexpected behavior.
- Hypothesize: Formulate potential causes.
- Test: Validate or refute the hypothesis using logs or a debugger.
- Fix: Implement the solution and verify it.
